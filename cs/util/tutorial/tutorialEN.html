Overview .:top:.

The IDE (Integrated Development Environment) of ClickScript consists of the following parts:
  • A : Menu Bar - Allows to show and hide parts of the IDE (e.g. Execution View, options and console).
  • B : Libraries - All loaded libraries with its components
  • C : Execution View - View for the execution
  • D : Actions and Status - Actions to run a clickscript as well as a runtime status
  • E : Programming View - Panel to click scripts together
  • F : Console - Console
Follow the tutorial to get more informations about this different parts.

Programming .:top:.

Basics

ClickScript is a simple to use visual programming language in the browser. A clickscript is a block diagram. The script consits of components and wires which connect the components. Each component has its colored input sockets on the left and its colored output sockets on the right side. On execution of a clickscript, a component gets executed as soon as all its inputs are ready. The different functionalities of a component are symbolized by the picture on it.

Components

Like already described, the components are the building blocks of ClickScript. You can add a component to the Programming View (E) by clicking on a component's icon in the library part (B). By moving the mouse pointer over a component in the Programming View, you get two icons: An information button (blue) and a remove button (red). To get a detailed description of a component, click onto the information button.
There exists three different types of components:

Primitives

Primitives are basic components to add values to the script. It is important to press ENTER or click again on the textfield after writing that the cursor disappears.

Modules

Modules are components which may consist of input and output sockets as well as fields.

Statements

Statements can have blocks like BODY in the upper example. These blocks can be resized by the buttons + and -. Components may be added to a block by simply moving the component onto/into the block.

Types

TEXT: any text
NUMBER: floating point number
LOGICAL VALUE: "true" / "false" or "1" / "0"
MINI-BROWSER: small browser window

Wiring Rules

Black dashed wires symbolize an invalid wiring. To remove a wire just double-click the wire.

Rule 1 - Same Types

Only connect sockets of the same color. The picture shows a violation of this rule.

Rule 2 - Input to Output

Always connect an output socket (a socket on the right side of a component) with an input socket (a socket on the left side of a component). The picture shows violation of this rule.

Rule 3 - One Wire per Input

Each input socket has exactly one wire connected. It is not allowed to have an unconnected input socket and it is not allowed to connect more than one wire to an input socket. There exists an exeption of this rule. If one wire is an outcoming wire of a TRUE block and the second connected wire to the input socket an outcoming wire of the FALSE block (of an IF/ELSE compnent), it is allowed to have two connected wires to the same input socket. The picture (a) shows such an exception of the rule.

Rule 4 - No Cyclic Graph

It is not allowed to build cyclic graphs. Examples like the ones given in the pictures are not allowed.
